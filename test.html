生成式 AI 比賽：從應用到 Agent

競賽分類與重點

競賽清單與技術核心：
  • LLM 應用開發組： 利用 LangChain 等框架製作自動化學習計畫生成器。
  • AI 影音藝術大賽： 考驗如何控制 Stable Diffusion 產出一致性高的影格。
  • Prompt 對抗賽： 利用 Chain-of-Thought (思維鏈) 引導 AI 通過邏輯測試。

RAG 技術應用

現代 AI 競賽的勝負手。選手需整合 Vector Database 與私有數據，解決 AI 幻覺問題。

貓咪盃：積木背後的 AI 代碼邏輯

AI 與 Scratch 的深度融合

  • ML 模型整合： 利用 Teachable Machine 訓練自定義模型並匯入 Scratch。
  • 電腦視覺感測： 透過鏡頭座標追蹤技術，實現手勢與角色的智慧互動。

競賽演進

已從純動畫轉向「智慧系統」。獲獎作品多具備 AI 感知能力，如：智慧動作矯正教練。

效能瓶頸

Scratch 在處理複雜卷積神經網絡時易發生延遲，需優化代碼邏輯與採樣頻率。

挑戰與限制：競賽發展的真實考驗

技術與資源不均

APCS 等高難度賽事極度依賴長期專業培訓，導致城鄉學生在資源獲取上存在明顯代溝。

硬體與成本門檻

GenAI 比賽需依賴高算力或付費 API (如 GPT-4)，開發成本常成為參賽者的隱形成障礙。

學習時間分配

競賽準備期長且強度高，如何平衡學科成績與競賽榮譽是青少年面臨的最大壓力。

原創性之定義

在 AI 輔助編寫代碼普及的時代，如何客觀評估作品的真實原創性，成為評審制度的新難題。
